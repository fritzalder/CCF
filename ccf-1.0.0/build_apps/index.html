Build Apps¶

This section describes how CCF applications can be developed and deployed to a CCF network.

Applications can be written in JavaScript/TypeScript or C++. An application consists of a collection of endpoints that can be triggered by Users. Each endpoint can define an API Schema to validate user requests.

These endpoints can read or mutate the state of a unique Key-Value Store that represents the internal state of the application. Applications define a set of Maps (see Creating a Map), mapping from a key to a value. When an application endpoint is triggered, the effects on the Store are committed atomically.
